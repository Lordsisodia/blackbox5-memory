## The Core Insight

**The Problem:** Current AI agents are ephemeral. They start fresh every session, losing all context, learnings, and improvements. They're tools, not collaborators. And there's no good harness for running them on real projects.

**BB5's Solution:** Create a system that:
1. **Runs Agents** - A harness for deploying AI agents on real tasks (not just BB5, but any project)
2. **Remembers** - Persistent memory across sessions (Hindsight Architecture)
3. **Learns** - Extract insights from every execution (RETAIN/RECALL/REFLECT)
4. **Improves** - Continuously optimize their own processes (RALF loops)
5. **Coordinates** - Multiple agents working together via queue-based coordination
6. **Navigates** - Self-orient in complex project hierarchies

## Key Architectural Principles

### 1. Self-Improvement as Core Mission
Not an afterthought. The system has goals about improving itself (CG-001: Continuous Self-Improvement).

### 2. Continuity of Self
The "Two Buffers" theory:
- **Buffer 1** (Functional): What we did (THOUGHTS.md, DECISIONS.md)
- **Buffer 2** (Subjective): Who we were, why we chose, how it felt (Hindsight memory)

### 3. Autonomy Through Constraints (Guidelines, Not Laws)
Strict rules enable autonomy, but they're flexible:
- **ONE task per run** - Main focus, but subtasks allowed within it
- **Documentation as needed** - 6 files is a guide, reduce if possible
- **Integration checks** - Code must work together
- **File-based communication** - Simple, no dependencies, debuggable

### 4. Calculated Priority
Priority isn't arbitrary. It's calculated:
```
priority = (impact / effort) * confidence
impact = (business_value * 0.4) + (technical_debt * 0.35) + (unblock_factor * 0.25)
```

### 5. Skill-Based Execution
Not generic tasks. Specialized skills matched to work:
- bmad-pm for product management
- bmad-architect for system design
- git-commit for git operations
- Auto-triggered based on task keywords

## Implications for Architecture Decisions

**When making decisions about BB5, ask:**

1. **Does this enable self-improvement?**
   - If yes, prioritize it
   - If no, question why it's needed

2. **Does this support continuity of self?**
   - Will agents remember this across sessions?
   - Is it captured in the memory system?

3. **Does this maintain autonomy?**
   - Can agents work without human intervention?
   - Are constraints clear and enforceable?

4. **Is the priority calculable?**
   - Can we measure impact, effort, confidence?
   - Does it fit the ROI model?

5. **Can skills handle this?**
   - Should this be a new skill?
   - Does it fit existing skill domains?
